                        <p class="text-gray-700 mb-3">执行 <code class="bg-gray-100 px-1 py-0.5 rounded">npm install</code> 时报错，提示“依赖安装失败”“超时”“包不存在”等。</p>  
                          
                        <h3 class="font-semibold text-lg mb-2">可能原因</h3>  
                        <ul class="list-disc pl-5 text-gray-700 mb-3 space-y-1">  
                            <li>Node.js 版本不兼容（需 18+）</li>  
                            <li>网络问题导致 npm 源无法访问</li>  
                            <li>缓存冲突</li>  
                        </ul>  
                          
                        <h3 class="font-semibold text-lg mb-2">解决方案</h3>  
                        <ol class="list-decimal pl-5 text-gray-700 mb-3 space-y-2">  
                            <li>  
                                检查 Node.js 版本：  
                                <div class="code-block mt-2">  
                                    <pre><code>node -v  # 需显示 v18.0.0 及以上</code></pre>  
                                </div>  
                                <p class="mt-2">若版本过低，前往 <a href="https://nodejs.org/" target="_blank" class="text-primary hover:underline">Node.js 官网</a> 下载安装 18+ 版本。</p>  
                            </li>  
                            <li>  
                                切换 npm 镜像源（解决网络问题）：  
                                <div class="code-block mt-2">  
                                    <pre><code>npm config set registry https://registry.npmmirror.com</code></pre>  
                                </div>  
                            </li>  
                            <li>  
                                清除 npm 缓存后重新安装：  
                                <div class="code-block mt-2">  
                                    <pre><code>npm cache clean --force  
rm -rf node_modules package-lock.json  # Windows 用：rd /s/q node_modules package-lock.json  
npm install</code></pre>  
                                </div>  
                            </li>  
                        </ol>  
                    </div>

                <section id="2" class="mb-8 scroll-mt-24">  
                    <h2 class="text-xl font-bold mb-4 text-primary flex items-center gap-2">  
                        <span class="bg-primary/10 text-primary p-1 rounded">2</span>  
                        端口占用无法启动  
                    </h2>  
                    <div class="pl-6 border-l-2 border-gray-200">  
                        <h3 class="font-semibold text-lg mb-2">问题现象</h3>  
                        <p class="text-gray-700 mb-3">执行 <code class="bg-gray-100 px-1 py-0.5 rounded">npm run start</code> 或 <code class="bg-gray-100 px-1 py-0.5 rounded">docker-compose up -d</code> 后，提示“Port 3000 is already in use”（端口 3000 被占用）。</p>  
                          
                        <h3 class="font-semibold text-lg mb-2">解决方案</h3>  
                        <ol class="list-decimal pl-5 text-gray-700 mb-3 space-y-2">  
                            <li>  
                                更换端口（推荐）：  
                                <ul class="list-disc pl-5 mt-2 space-y-1">  
                                    <li>本地部署：在根目录 <code class="bg-gray-100 px-1 py-0.5 rounded">.env</code> 文件中添加 <code class="bg-gray-100 px-1 py-0.5 rounded">PORT=8080</code>（可替换为 8081、3001 等未占用端口），重新启动。</li>  
                                    <li>Docker 部署：修改 <code class="bg-gray-100 px-1 py-0.5 rounded">docker-compose.yml</code> 中 <code class="bg-gray-100 px-1 py-0.5 rounded">ports</code> 配置，例如 <code class="bg-gray-100 px-1 py-0.5 rounded">8080:3000</code>（左侧为本地端口，右侧为容器端口）。</li>  
                                </ul>  
                            </li>  
                            <li>  
                                关闭占用端口的进程：  
                                <div class="grid grid-cols-1 md:grid-cols-2 gap-4 mt-2">  
                                    <div>  
                                        <p class="font-medium">Windows（PowerShell）：</p>  
                                        <div class="code-block mt-1">  
                                            <pre><code>netstat -ano | findstr "3000"  # 找到占用进程的 PID（最后一列数字）  
taskkill /F /PID 进程PID</code></pre>  
                                        </div>  
                                    </div>  
                                    <div>  
                                        <p class="font-medium">Mac/Linux：</p>  
                                        <div class="code-block mt-1">  
                                            <pre><code>lsof -i :3000  # 找到占用进程的 PID  
kill -9 进程PID</code></pre>  
                                        </div>  
                                    </div>  
                                </div>  
                            </li>  
                        </ol>  
                    </div>  
                </section>  
  
                <!-- 3. 访问不了网站 -->  
                <section id="3" class="mb-8 scroll-mt-24">  
                    <h2 class="text-xl font-bold mb-4 text-primary flex items-center gap-2">  
                        <span class="bg-primary/10 text-primary p-1 rounded">3</span>  
                        访问不了网站  
                    </h2>  
                    <div class="pl-6 border-l-2 border-gray-200">  
                        <h3 class="font-semibold text-lg mb-2">问题现象</h3>  
                        <p class="text-gray-700 mb-3">部署后访问 <code class="bg-gray-100 px-1 py-0.5 rounded">http://localhost:3000</code> 或服务器 IP+端口，提示“无法访问此网站”“超时”“404”。</p>  
                          
                        <h3 class="font-semibold text-lg mb-2">可能原因</h3>  
                        <ul class="list-disc pl-5 text-gray-700 mb-3 space-y-1">  
                            <li>服务未启动成功</li>  
                            <li>防火墙/安全组未开放端口</li>  
                            <li>服务器 IP 或端口配置错误</li>  
                        </ul>  
                          
                        <h3 class="font-semibold text-lg mb-2">解决方案</h3>  
                        <ol class="list-decimal pl-5 text-gray-700 mb-3 space-y-2">  
                            <li>  
                                检查服务状态：  
                                <ul class="list-disc pl-5 mt-2 space-y-1">  
                                    <li>本地部署：执行 <code class="bg-gray-100 px-1 py-0.5 rounded">npm run start</code> 后，查看终端是否显示“Server started on port 3000”（无报错）。</li>  
                                    <li>Docker 部署：执行 <code class="bg-gray-100 px-1 py-0.5 rounded">docker ps</code>，查看容器是否处于“Up”状态（若为“Exited”，执行 <code class="bg-gray-100 px-1 py-0.5 rounded">docker logs 容器名</code> 查看报错日志）。</li>  
                                </ul>  
                            </li>  
                            <li>  
                                开放端口：  
                                <ul class="list-disc pl-5 mt-2 space-y-1">  
                                    <li>本地：关闭电脑防火墙（临时测试），或在防火墙设置中允许端口 3000 访问。</li>  
                                    <li>服务器（阿里云/腾讯云等）：在安全组规则中添加“入方向”，允许端口 3000（或自定义端口）的 TCP 访问。</li>  
                                </ul>  
                            </li>  
                            <li>  
                                确认访问地址：  
                                <ul class="list-disc pl-5 mt-2 space-y-1">  
                                    <li>本地部署：仅能通过 <code class="bg-gray-100 px-1 py-0.5 rounded">http://localhost:3000</code> 访问（同一局域网内可通过电脑 IP+端口访问）。</li>  
                                    <li>服务器部署：需通过服务器公网 IP+端口访问（例如 <code class="bg-gray-100 px-1 py-0.5 rounded">http://123.45.67.89:3000</code>）。</li>  
                                </ul>  
                            </li>  
                        </ol>  
                    </div>  
                </section>  
  
                <!-- 4. 数据库连接失败 -->  
                <section id="4" class="mb-8 scroll-mt-24">  
                    <h2 class="text-xl font-bold mb-4 text-primary flex items-center gap-2">  
                        <span class="bg-primary/10 text-primary p-1 rounded">4</span>  
                        数据库连接失败  
                    </h2>  
                    <div class="pl-6 border-l-2 border-gray-200">  
                        <h3 class="font-semibold text-lg mb-2">问题现象</h3>  
                        <p class="text-gray-700 mb-3">启动服务后，提示“Failed to connect to database”“数据库连接超时”。</p>  
                          
                        <h3 class="font-semibold text-lg mb-2">可能原因</h3>  
                        <ul class="list-disc pl-5 text-gray-700 mb-3 space-y-1">  
                            <li>数据库类型配置错误</li>  
                            <li>数据库地址/账号密码错误</li>  
                            <li>数据库未启动（Docker 部署）</li>  
                        </ul>  
                          
                        <h3 class="font-semibold text-lg mb-2">解决方案</h3>  
                        <ol class="list-decimal pl-5 text-gray-700 mb-3 space-y-2">  
                            <li>  
                                检查环境变量配置（<code class="bg-gray-100 px-1 py-0.5 rounded">.env</code> 文件）：  
                                <ul class="list-disc pl-5 mt-2 space-y-1">  
                                    <li>使用 SQLite（默认）：确保 <code class="bg-gray-100 px-1 py-0.5 rounded">DATABASE_TYPE=sqlite</code>，无需额外配置 <code class="bg-gray-100 px-1 py-0.5 rounded">DATABASE_URL</code>。</li>  
                                    <li>使用 MySQL：确保 <code class="bg-gray-100 px-1 py-0.5 rounded">DATABASE_URL=mysql://用户名:密码@数据库地址:3306/数据库名</code>，且 MySQL 服务已启动、账号密码正确。</li>  
                                </ul>  
                            </li>  
                            <li>  
                                Docker 部署 MySQL 时：  
                                <ul class="list-disc pl-5 mt-2 space-y-1">  
                                    <li>执行 <code class="bg-gray-100 px-1 py-0.5 rounded">docker ps</code> 检查 <code class="bg-gray-100 px-1 py-0.5 rounded">db</code> 容器是否启动，若未启动，执行 <code class="bg-gray-100 px-1 py-0.5 rounded">docker start db</code>。</li>  
                                    <li>检查 <code class="bg-gray-100 px-1 py-0.5 rounded">docker-compose.yml</code> 中 <code class="bg-gray-100 px-1 py-0.5 rounded">DATABASE_URL</code> 是否与 MySQL 容器配置一致（例如用户名、密码、数据库名）。</li>  
                                </ul>  
                            </li>  
                        </ol>  
                    </div>  
                </section>

                        <span class="bg-primary/10 text-primary p-1 rounded">6</span>  
                        资料无法自动更新  
                    </h2>  
                    <div class="pl-6 border-l-2 border-gray-200">  
                        <h3 class="font-semibold text-lg mb-2">问题现象</h3>  
                        <p class="text-gray-700 mb-3">后台未按设定的 Cron 表达式更新资料，时政热点、真题未同步。</p>  
                          
                        <h3 class="font-semibold text-lg mb-2">可能原因</h3>  
                        <ul class="list-disc pl-5 text-gray-700 mb-3 space-y-1">  
                            <li>自动更新功能未启用</li>  
                            <li>Cron 表达式格式错误</li>  
                            <li>网络问题导致无法获取更新资源</li>  
                        </ul>  
                          
                        <h3 class="font-semibold text-lg mb-2">解决方案</h3>  
                        <ol class="list-decimal pl-5 text-gray-700 mb-3 space-y-2">  
                            <li>检查环境变量：确保 <code class="bg-gray-100 px-1 py-0.5 rounded">.env</code> 文件中 <code class="bg-gray-100 px-1 py-0.5 rounded">UPDATE_CRON</code> 格式正确（例如 <code class="bg-gray-100 px-1 py-0.5 rounded">0 3 * * *</code> 表示每日凌晨 3 点）。</li>  
                            <li>  
                                手动触发更新（测试）：  
                                <ul class="list-disc pl-5 mt-2 space-y-1">  
                                    <li>本地部署：执行 <code class="bg-gray-100 px-1 py-0.5 rounded">npm run update</code> 手动触发资料更新。</li>  
                                    <li>Docker 部署：执行 <code class="bg-gray-100 px-1 py-0.5 rounded">docker exec -it 169 npm run update</code>（169 为容器名称，可通过 <code class="bg-gray-100 px-1 py-0.5 rounded">docker ps</code> 查看）。</li>  
                                </ul>  
                            </li>  
                            <li>检查网络：确保部署环境能正常访问外网（需获取时政、真题等网络资源）。</li>  
                        </ol>  
                    </div>
